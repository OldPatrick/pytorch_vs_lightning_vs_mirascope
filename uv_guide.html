    Here's the updated guide with custom virtual environment options added:

    1. Download and Manage Python with uv

    A. Download a Specific Python Version

    Bash
    uv python install 3.11

    This downloads and installs Python 3.11 to uv's managed Python directory.

    B. List Available/Installed Python Versions

    Bash
    uv python list

    Shows both available and installed Python versions.

    2. Standard Project Setup with uv

    A. Create and Navigate to Project Directory

    Bash
    mkdir myproject
cd myproject

    B. Initialize the Project

    Bash
    uv init

    Creates a pyproject.toml with basic project structure.

    C. Pin the Python Version

    Since you've already downloaded Python 3.11 with uv:

    Bash
    uv python pin 3.11

    This:

    • Sets Python 3.11 for this project
    • Updates requires-python in pyproject.toml
    • Creates/updates .python-version file

    D. Create Virtual Environment

    Default .venv:

    Bash
    uv venv

    Custom Environment Name/Location:

    Bash
    # Create environment with custom name in current directory
uv venv myenv

# Create environment in specific location
uv venv ~/environments/myproject-env

# Create environment with specific Python version
uv venv --python 3.11 custom-env

    E. Activate the Virtual Environment

    For Default .venv:

    Bash
    # Linux/macOS
source .venv/bin/activate

# Windows
.venv\Scripts\activate

    For Custom Environment:

    Bash
    # Linux/macOS
source myenv/bin/activate
# or for specific path
source ~/environments/myproject-env/bin/activate

# Windows
myenv\Scripts\activate
# or for specific path
~\environments\myproject-env\Scripts\activate

    F. Tell uv to Use Your Custom Environment

    If you want uv commands to use a non-default environment:

    Bash
    # Set UV_PROJECT_ENVIRONMENT variable
export UV_PROJECT_ENVIRONMENT=myenv

# Or use --python flag with each command
uv add --python myenv requests

    G. Add Dependencies

    Bash
    uv add requests pandas numpy

    This:

    • Adds packages to pyproject.toml
    • Updates uv.lock
    • Installs packages in the active environment

    3. Working with Multiple Environments

    A. Create Multiple Environments for Different Purposes

    Bash
    # Development environment
uv venv dev-env
source dev-env/bin/activate
uv add --dev pytest black ruff

# Production environment
uv venv prod-env
source prod-env/bin/activate
uv sync --no-dev  # Only production dependencies

# Testing environment with different Python
uv venv --python 3.12 test-env
source test-env/bin/activate
uv sync

    B. Switch Between Environments

    Bash
    # Deactivate current
deactivate

# Activate different one
source prod-env/bin/activate

    4. Working with Existing pyproject.toml Files

    A. If You Have a uv/PEP 621 Compatible pyproject.toml

    1. Create and activate custom environment:

      Bash
      uv venv myproject-env
source myproject-env/bin/activate  # Linux/macOS

    2. Pin Python version (if not already specified):

      Bash
      uv python pin 3.11

    3. Install all dependencies:

      Bash
      uv sync

    B. If You Have a Poetry pyproject.toml

    Poetry uses a different format ([tool.poetry.dependencies]). You have two options:

    Option 1: Convert to PEP 621 Format

    Manually convert:

    toml
    # Poetry format
[tool.poetry.dependencies]
python = "^3.11"
requests = "^2.31.0"
pandas = "^2.0.0"

# Convert to PEP 621 format
[project]
requires-python = ">=3.11"
dependencies = [
    "requests>=2.31.0",
    "pandas>=2.0.0"
]

    Then run:

    Bash
    uv venv custom-env
source custom-env/bin/activate
uv sync

    Option 2: Fresh Start with uv

    1. Note your dependencies
    2. Initialize fresh with uv init
    3. Create custom environment: uv venv myenv
    4. Activate it: source myenv/bin/activate
    5. Pin Python: uv python pin 3.11
    6. Add dependencies: uv add requests pandas

    Complete Example Workflow with Custom Environment

    Bash
    # 1. Download Python if needed
uv python install 3.11

# 2. Create project
mkdir awesome-project
cd awesome-project

# 3. Initialize
uv init

# 4. Pin Python version
uv python pin 3.11

# 5. Create custom virtual environment
uv venv project-env

# 6. Activate the environment
source project-env/bin/activate  # Linux/macOS
# or
project-env\Scripts\activate  # Windows

# 7. Add dependencies
uv add fastapi uvicorn pytest

# 8. Add dev dependencies
uv add --dev black ruff mypy

# 9. Run commands in the environment
python app.py
# or without activation
uv run --python project-env python app.py

    Absolutely! Here’s a clear guide for using the uv package manager to set up a new Python project, specify a Python version, and handle dependencies—including if you already have a pyproject.toml file (from uv or poetry).

    1. Standard Setup with uv

    A. Create a New Project Directory

    Bash
    mkdir myproject
cd myproject

    B. Initialize a New Project with uv

    This creates a pyproject.toml compatible with uv (and PEP 621).

    Bash
    uv init

    You’ll be prompted for project details (name, version, etc.).

    C. Specify a Python Version

    Edit the generated pyproject.toml and add (or modify) the requires-python field under [project]:

    toml
    [project]
name = "myproject"
version = "0.1.0"
requires-python = ">=3.10,<3.13"

    Note: uv does not manage Python installations, but this field ensures the environment uses the correct version.

    D. Add Dependencies

    To add packages:

    Bash
    uv pip install requests flask

    This updates your pyproject.toml and uv.lock with the dependencies.

    2. If You Already Have a pyproject.toml

    A. If It’s a uv-style or PEP 621 pyproject.toml

    Just run:

    Bash
    uv pip install

    This will install all dependencies listed in [project] and [project.optional-dependencies].

    B. If It’s a Poetry-style pyproject.toml

    • uv supports PEP 621, which is also used by Poetry, but some Poetry-specific features (like [tool.poetry.*]) may not be fully compatible.
    • If your pyproject.toml uses [tool.poetry.dependencies], you may need to convert it to the PEP 621 [project] format for best compatibility with uv.

    Convert Poetry to PEP 621 (if needed):

    • Move dependencies from [tool.poetry.dependencies] to [project] as dependencies = [...].
    • Example: 
      toml
      [project]
dependencies = [
    "requests>=2.0",
    "flask"
]

    Then run:

    Bash
    uv pip install

    Install from Poetry pyproject.tomlConvert to PEP 621, then uv pip installPoetry-specific fields may need conversion

    Tip:

    • uv is fast and compatible with most modern Python packaging standards, but always check your pyproject.toml for compatibility if migrating from Poetry.
